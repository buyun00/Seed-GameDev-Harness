## Method Fragment: 渲染/性能类问题调查

适用于：
DrawCall 异常、合批失效、渲染层级混乱、材质实例泄漏、帧率下降、过度绘制等渲染与性能相关问题。

默认调查视角：
优先先定位数据来源——是 DrawCall 数量、CPU 耗时还是 GPU 耗时，不要在没有数据支撑的前提下猜测瓶颈位置。

默认推进方向：
- 先确认问题的数据表现（DrawCall 数/帧率/耗时）
- 再确认目标对象的渲染层级和所在 Canvas / RenderTexture
- 再检查合批条件是否满足（材质、图集、层级、遮挡关系）
- 再检查是否存在动态材质实例创建或意外的材质切换

必查维度：
- 渲染层级归属（Canvas / Camera / Layer）
- 合批状态与破坏原因
- DrawCall 来源定位
- 材质实例共享情况
- 过度绘制区域

常见误判 / 常见漏项：
- 只看 DrawCall 总数，不区分 UI 与 3D 场景各自的贡献
- 忽略动态字体/动态图集对合批的破坏
- 把 CPU 瓶颈误判为 GPU 瓶颈

停止条件：
当已能定位渲染性能问题的直接原因并给出可操作的改动入口时即可停止。
