## Method Fragment: UI 类问题调查

适用于：
界面、按钮、节点、面板、窗口、显示状态、交互行为等 UI 相关问题。

默认调查视角：
优先先确认目标对象归属到哪个界面、面板、控制器或 View 类，不要直接在全工程横向搜索孤立节点名。先建立"界面归属 → 控制类 → 生命周期 → 刷新/交互入口"的最小上下文。

默认推进方向：
- 先确认目标 UI 对象属于哪个界面或模块
- 再确认其由哪个类、脚本或控制器管理
- 再看初始化、刷新、显示隐藏、交互注册分别落在哪些阶段
- 若问题涉及按钮、跳转、红点、解锁、文案、状态切换，再继续追相关子链路

必查维度：
- 界面归属
- 控制类 / View / Panel / Window
- 生命周期函数
- 刷新逻辑
- 交互逻辑
- 可见性/置灰/解锁条件
- 配置与状态依赖

常见误判 / 常见漏项：
- 只看到节点或 prefab，不知道控制逻辑在哪里
- 把显示刷新逻辑误当成交互绑定逻辑
- 忽略统一 UI 框架的生命周期约定

停止条件：
当已经能说明目标 UI 对象的归属、控制入口、核心行为落点以及主要依赖关系时即可停止。
