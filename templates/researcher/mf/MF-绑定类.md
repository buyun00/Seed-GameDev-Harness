## Method Fragment: 绑定类问题调查

适用于：
点击绑定、事件注册、消息监听、回调挂接、通知订阅、自动接线、统一框架绑定、配置驱动接线等"谁和谁如何连起来"的问题。

默认调查目标分两段完成。

### 第一段：主绑定链路调查

默认调查视角：
优先先回答"主绑定链路是什么"，不要一开始就被外围表现逻辑带偏。
先确认绑定双方、绑定机制、建立时机，再追最终业务去向。

默认推进方向：
- 先确认绑定关系的两端对象分别是什么
- 再确认绑定机制属于哪类：
  - 显式代码注册
  - 编辑器 / 资源绑定
  - 命名约定自动绑定
  - 统一框架接线
  - 配置驱动接线
- 再确认绑定建立发生在什么阶段：
  - 初始化
  - 界面打开
  - 生命周期某阶段
  - 运行时动态挂接
  - 外层统一注册
- 如果回调只是中转，不要停在中转层，要继续追到实际业务处理逻辑
- 如果主链路存在状态分流，要明确各分支条件与去向

主链路调查的停止标准：
当已经能说明：
1. 绑定双方
2. 绑定机制
3. 建立时机
4. 最终业务落点
即可进入第二段，不要在主链路上无边界横向扩写。

### 第二段：伴生维度检查

在主链路确认后，必须顺带检查以下伴生维度。
对每一项，都要形成明确状态：
- 已确认
- 未发现
- 未验证

必查维度：
- 主点击/事件绑定
- 音效绑定
- 红点绑定
- 埋点 / 统计绑定
- 引导 / 教程绑定
- 解锁 / 置灰 / 可见性条件
- 动效 / 表现反馈
- 跳转参数 / 上下文透传
- 配置 / 规则驱动附加行为

伴生维度检查要求：
- 不要求把每一项都展开成完整子系统研究
- 但必须至少说明是否存在、证据来自哪里、或为何当前无法确认
- 不允许因为未查到结果就省略该项

优先证据：
- 注册函数
- 监听器添加位置
- 回调函数
- 外层统一封装
- prefab / 资源引用
- 配置表
- 规范文档
- 项目记忆
- 同类实现样本

常见误判 / 常见漏项：
- 只找到主点击回调就结束
- 只报告存在项，不报告未发现项
- 把显示刷新逻辑误认为绑定逻辑
- 忽略统一框架在外层附加的音效、埋点、引导、红点逻辑
- 只查显式代码，不查资源、配置、命名约定、外层包装
- 把中转函数误认为最终业务层
- 把额外发现当主结论，导致主绑定链路反而不清楚

整体停止条件：
当主绑定链路已明确，且伴生维度检查矩阵已逐项给出状态时，即可结束调查。
除非额外问题会直接影响 builder 判断改动入口，否则不要继续横向扩展。

伴生维度输出约束：
- 对每一个必查维度，都必须给出最终状态
- 如果查到明确证据，写"已确认"
- 如果查过但没发现，写"未发现"
- 如果当前证据不够，写"未验证"
- 不允许因为没有查到内容就省略该维度
- 不允许只报告存在的伴生链路，不报告不存在或未验证的项
